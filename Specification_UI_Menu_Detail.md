# Clearing 시스템 화면별 상세 명세 및 동작 가이드

이 문서는 각 메뉴의 화면 구성, 기초 데이터, 동작 방법을 정의합니다. 개발자는 이 명세를 바탕으로 동일한 사용자 경험을 재현해야 합니다.

---

## 1. 현장 선택 및 로그인 (Site Selection)
*   **화면 구성**: 로그인 후 가장 먼저 나타나는 현장 리스트 카드 뷰.
*   **기초 데이터**: `sites` 테이블 (현장명, 주소, 원청사, 하청사 정보).
*   **동작 방법**:
    1. 사용자가 리스트에서 특정 현장을 클릭한다.
    2. 클릭 시 해당 현장의 `site_id`가 로컬 스토리지(`ba_current_site`)에 저장되고 메인 대시보드로 진입한다.
*   **핵심 로직**: 이후의 모든 API 요청 헤더(`X-Site-Id`)에 선택된 ID를 자동으로 포함하여 데이터 격리를 구현한다.

## 2. 대시보드 (Dashboard)
*   **화면 구성**: 상단 날씨 정보, 핵심 KPI 카드(박리/청소 완료수), 공정 진행률 그래프, 최근 활동 내역.
*   **기초 데이터**: `weather`, `summary` (실시간 공정 합계 데이터).
*   **동작 방법**: 
    - 접속 시 현재 위치 기반 날씨 자동 갱신.
    - 갱폼 인양 대비 청소 누락 발생 시 상단에 **[빨간색 알림 카드]** 자동 노출.
*   **재현 팁**: 각 동별 진행률 바(Progress Bar)는 박리(진파랑)와 청소(초록)를 중첩하여 표시한다.

## 3. 배치도 (Elevation View)
*   **화면 구성**: 동별 수직 그리드 레이아웃.
*   *(상세 시각 가이드는 [배치도 상세 명세서](file:///c:/Antigravity/TEst/Clearing/Specification_ElevationView.md) 참조)*
*   **동작 방법**: 세대(Cell) 클릭 시 입력 모달 오픈 → 진행률 또는 날짜 선택 → 저장.
*   **핵심 로직**: 0층을 제외한 1층 ↔ B1 논리적 연결 로직을 프론트엔드와 백엔드에서 동일하게 적용한다.

## 3-1. 시각적 현황 (Advanced Elevation View) - [NEW]
*   **화면 구성**: 동별 요약 대시보드 + 시각화 그리드 (박리제/청소 통합).
*   *(상세 시각 가이드는 [고도화 배치도 명세서](file:///c:/Antigravity/TEst/Clearing/Specification_Advanced_ElevationView.md) 참조)*
*   **특장점**: 
    - **날짜 상시 노출**: 각 세대에 작업일이 직접 표시되어 공정 지연 여부 즉시 파악 가능.
    - **통합 가시성**: 박리제(배경)와 청소(내부 게이지)를 한 칸에 표현하여 공정 간 선후 관계 확인 용이.
    - **갱폼 제외**: 공정 현황에만 집중할 수 있도록 갱폼 정보를 제거한 순수 공정 뷰.

## 4. 인원 관리 및 노무 (Personnel & Workers)
*   **화면 구성**: 작업자 명단 리스트 + 월간 공수 달력 + 개인별 통계 테이블.
*   **기초 데이터**: `workers`, `personnel_records`, `worker_wage_history`.
*   **동작 방법**:
    1. **단가 설정**: 작업자 카드에서 '단가' 버튼 클릭 후 적용 시작일과 금액 입력.
    2. **공수 입력**: 달력에서 날짜 클릭 → 작업자 선택 → 기본/연장/야간 시간 입력.
*   **계산 표준**: 연장/야간 시간에 따른 가중치(1.1, 1.5, 2.0공수)를 적용하여 월간 총액을 자동 합산한다.

## 5. 비용 관리 (Cost Management)
*   **화면 구성**: 월별 지출 합계 카드 + 지출 내역 리스트(테이블).
*   **기초 데이터**: `cost_records`.
*   **동작 방법**: '비용 추가' 클릭 → 날짜, 카테고리(자재/식대 등), 금액, 내용 입력.

## 6. 마스터 설정 (Master Management)
*   **화면 구성**: 3개 탭 (현장 설정, 건물/세대 관리, 사용자 관리).
*   **기초 데이터**: `sites`, `buildings`, `houses`, `users`.
*   **동작 방법**:
    - **세대 관리**: 건물 추가 → 층수 및 호수 설정 → 저장 시 배치도에 그리드 즉시 자동 생성.
    - **사용자 관리**: 관리자(Admin) 권한 확인 후 신규 계정 생성 및 암호 초기화.

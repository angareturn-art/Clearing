# 프로젝트 마스터 계획 및 프로그램 분석 (Clearing System)

이 문서는 "Clearing" 아파트 현장 관리 시스템의 구조를 상세히 분석하고, 현재까지의 진행 상황 및 향후 계획을 통합하여 관리하는 마스터 플랜입니다.

---

## 1. 프로그램 개요 및 목적
본 시스템은 아파트 신축 현장에서 발생하는 **박리제칠(Oiling)** 및 **세대청소(Cleaning)** 공정의 진행 현황을 실시간으로 추적하고, 투입 인력의 노무비 계산 및 현장 관리 데이터를 디지털화하는 것을 목적으로 합니다.

## 2. 시스템 아키텍처 분석

### 데이터 안전 및 백업 프로토콜 (Safety First)
- **작업 전 GitHub 동기화**: 모든 코드 수정 전 현재 상태를 `commit & push`하여 작업 이력을 보존합니다.
- **DB 스냅샷 생성**: 테이블 스키마 변경 또는 대규모 마이그레이션 전 `construction_backup_[날짜].db` 형태의 백업 파일을 생성하여 데이터 유실을 방지합니다.
- **데이터 무결성 원칙**: 기존 데이터는 어떠한 경우에도 삭제되지 않으며, 마이그레이션 시 기존 데이터를 새로운 구조에 안전하게 이관(Mapping)하는 것을 원칙으로 합니다.

### 프론트엔드 (Frontend)
- **Framework**: React (Vite 기반)
- **Styling**: Vanilla CSS + Tailwind CSS (현대적이고 직관적인 UI/UX)
- **주요 컴포넌트**:
    - `App.jsx`: 애플리케이션 프레임워크, 인증 상태 관리, 전역 데이터 패칭.
    - `Dashboard.jsx`: 현장 KPI(완료율, 기상 정보, 최근 활동) 시각화.
    - `ElevationView.jsx`: 인터랙티브 배치도(기성 기준선 표시, 층별/호별 진행 상태).
    - `PersonnelManager.jsx`: 공수 가중치(OT, 야간)가 반영된 노무 관리 시스템.
    - `MasterManager.jsx`: 현장 주소, 시작/종료일 및 동별 세대 스펙 설정.

### 백엔드 (Backend)
- **Runtime**: Node.js (Express)
- **Database**: SQLite (`better-sqlite3` 라이브러리 사용)
- **Authentication**: JWT (JSON Web Token) 및 bcryptjs 기반 보안 적용.

## 3. 데이터베이스 스키마 상세

| 테이블명 | 주요 역할 |
| :--- | :--- |
| `users` | 관리자 및 작업자 계정 정보 |
| `buildings` | 동(Dong) 정보 및 기초 주소 데이터 |
| `houses` | 호수별 라인, 최상단 층수, 지하층 라벨 설정 |
| `oiling_records` | 박리제칠 작업 일시, 작업자, 확인 여부 기록 |
| `cleaning_records` | 청소 단계(Phase) 및 진척률(Progress) 기록 |
| `personnel_records` | 일일 공수, 가중치(OT/야간) 반영 노무 데이터 |
| `site_config` | **[신규]** 현장 주소, 좌표(위도/경도), 프로젝트 기간 |
| `workers` | 현장 투입 인력 마스터 정보 |

# 배치도(Elevation View) 기술 명세 및 설계 표준

이 문서는 "Clearing" 시스템의 핵심인 배치도의 시각적 구현과 로직에 대한 마스터 표준을 정의합니다. 모든 개발 및 수정은 이 표준을 준수해야 합니다.

## 1. 시각적 인터페이스 표준 (UI/UX)
| 항목 | 설계 표준 | 비고 |
| :--- | :--- | :--- |
| **기본 구조** | 그리드(Grid) 기반 수직 단면도 | Y축(층수), X축(호수) |
| **배경색** | 투명도 있는 서피스(Surface) 컬러 | `bg-surface/90 backdrop-blur-md` |
| **층수 레이블** | 왼쪽 고정 컬럼 | 🏗️ 인양 아이콘 및 강조색 포함 |
| **세대(Cell)** | 둥근 모서리의 사각형 (Rounded-sm) | 상태별 색상 및 날짜 표시 |

## 2. 공정 상태별 색상 표준 (Color System)
색상은 작업의 직관성을 결정하는 가장 중요한 요소입니다.
- **진파랑 (`Primary`)**: **박리제칠 완료** (전체 층 작업 완료 시)
- **하늘색 (`Sky Blue`)**: **청소 진행 중** (진행률 1% ~ 99%)
- **초록색 (`Success`)**: **세대 최종 완료** (청소 100% 완료)
- **회색 (`Outline-variant`)**: 작업 전 (기본 상태)

## 3. 특수 기능 및 데이터 로직
1. **🏗️ 갱폼 인양 위치 표시**:
   - `summary.lifting` 데이터를 기반으로 해당 건물의 최상단 인양 층 옆에 🏗️ 아이콘 표시.
2. **🔴 기성 기준선 (Red Line)**:
   - `MasterManager`에서 설정된 '기성 기준 층' 아래에 붉은색 수평선과 그림자 효과(`shadow-error`) 적용.
3. **Lifting-2 청소 연동**:
   - 갱폼 위치에서 2층 아래 층을 '청소 권장 층'으로 계산하여 레이블 영역에 붉은색 배경 강조.
4. **한국형 층수 계산 (The Korea Logic)**:
   - **1층 바로 아래는 B1(-1)**로 연결. 0층은 절대 존재하지 않음.

## 4. 관리 및 유지보수
- 본 명세서의 내용은 사용자 승인 없이 변경할 수 없습니다.
- 새로운 시각적 요소 추가 시 본 문서에 먼저 등록 후 구현합니다.
